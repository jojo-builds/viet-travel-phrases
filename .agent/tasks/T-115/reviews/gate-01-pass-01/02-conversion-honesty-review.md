APPROVE

## Evidence
The current Vietnam country-page copy is explicit that the site is only a starter preview and that the app carries the deeper backup. Both the flat page and routed page say the website surfaces selected starter phrases/modules, not the whole app library, and both are identical, so the honesty claim is consistent across the route pair.

## Risks
The line about "150 starter entries, 750 premium entries, and 900 total visible entries" could be skimmed as a website-surface claim if a reader ignores the follow-up sentence that says this is library depth, not full web exposure. That is a wording risk, but the surrounding copy already corrects it.

## Recommendation
Keep the current framing. It is honest about the site/app boundary, matches the approved starter-subset posture, and does not claim full-library web coverage.
